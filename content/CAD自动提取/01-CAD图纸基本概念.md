# 01- CAD 图纸基本概念

> 本节目标:建立 CAD 自动提取的核心心智模型——CAD 图本质上是什么、实体/图层/块是什么、坐标系与单位有哪些坑。后续所有代码都建立在这些概念上。

## 一、CAD 图的本质

**关键事实**:CAD 图**不是图片**(不是 PNG/JPG 那种由像素组成的位图),而是**一堆带数学定义的几何对象**。

举例:一条从 (0, 0) 到 (100, 50) 的直线,文件里实际记录的是:

```
类型: LINE
起点: (0, 0)
终点: (100, 50)
图层: 巷道中线
颜色: 红色
线型: 实线
```

显示时由 CAD 软件按几何参数**临时渲染**给你看。

### 对自动提取的意义

因为 CAD 图本质是**结构化几何数据**,所以可以**直接"读"**,不需要图像识别、不需要 OCR、不需要训练模型。

| | 数据形式 | 怎么"读" |
|---|---|---|
| 扫描的纸图(PNG) | 一堆像素 | 必须图像识别/OCR/矢量化(难) |
| CAD 图(DWG/DXF) | 一堆几何对象的结构化描述 | 直接按格式解析(相对简单) |

## 二、实体(Entity)——代码里的主角

> **实体 = 一个独立的、可见的图形单元。**

CAD 图里所有可见的东西都是实体:LINE、CIRCLE、ARC、LWPOLYLINE、TEXT、MTEXT、INSERT、DIMENSION 等等。

### 实体的三件套结构

```
┌─────────────────────────────────────┐
│  实体(Entity)                       │
├─────────────────────────────────────┤
│  1. 类型      LINE / CIRCLE / TEXT…  │
│  2. 几何参数  起点、终点、半径、文字…  │
│  3. 公共属性  图层、颜色、线型、线宽   │
└─────────────────────────────────────┘
```

口诀:**几何参数 = "确定这个形状所需要的最少信息"**。

### 常见实体的几何参数

| 实体类型 | 几何参数 |
|---|---|
| LINE(直线) | 起点 (x, y, z)、终点 (x, y, z) |
| CIRCLE(圆) | 圆心、半径 |
| ARC(弧) | 圆心、半径、起始角、结束角 |
| LWPOLYLINE(多段线) | **一串顶点** [(x1, y1), (x2, y2), ...] |
| TEXT(单行文字) | 插入点、文字内容、字高、旋转角 |
| INSERT(块引用) | 插入点、引用的块名、缩放、旋转 |

### 自动提取的核心动作

```
第 1 步:遍历图里所有实体
第 2 步:对每个实体,按"类型 + 图层 + 几何参数"做筛选和处理
```

## 三、图层(Layer)——主过滤维度

### 直觉

像透明胶片叠加:每张胶片只画一类东西(道路/河流/建筑/注记),叠起来就是完整地图。

**每个实体都"属于"某一个图层。**

### 图层的作用

1. **分类组织**——按性质把实体分门别类
2. **批量控制**——隐藏/显示、统一改颜色、统一打印
3. **过滤提取**(⭐ 对我们最重要)

### 煤矿图常见图层(示例)

| 图层名 | 装什么 |
|---|---|
| 巷道中线 | 巷道的中心线(LWPOLYLINE) |
| 巷道边线 | 巷道的两侧轮廓线 |
| 断层 | 断层线、断层标注 |
| 钻孔 | 钻孔符号(INSERT) |
| 地层界线 | 地层分界线 |
| 注记 | 文字标注 |
| 图框 | 图纸外框、标题栏 |

### 关键事实

- **图层名是字符串,由画图人自己起的**——同类东西在不同图里可能叫不同名字
- 接到新图永远先做**"侦察"**:列出所有图层、搞清楚每个图层装的是什么
- **图层 + 实体类型 两个条件组合,能筛掉 90% 不相关的东西**

## 四、块(Block)——可复用的图形组件

### 直觉:图章机制

画 200 个一样的阀门符号? 画一次定义成"图章",在 200 个位置"盖章"。

### 两个关键术语

**块定义(Block Definition)**:图章本身
- 内部由若干基础实体组成(如阀门符号 = 1 圆 + 2 线 + 1 文字)
- 有一个名字,如 `VALVE`、`钻孔`、`ZK`

**块引用(Block Reference / INSERT)**:盖在纸上的一个印记
- 本身是个实体,类型叫 **INSERT**
- 只记录:引用的块名 + 插入位置 + 缩放 + 旋转

### 关系图

```
                 块定义 "钻孔"
                 (内部:1 个圆 + 1 段文字)
                       │
        ┌──────────────┼──────────────┐
        │              │              │
   INSERT             INSERT         INSERT
   位置(100,200)    位置(350,180)  位置(500,150)
   缩放 1.0          缩放 0.8       缩放 1.0
   旋转 0°           旋转 0°        旋转 45°
```

→ 一个块定义,多个 INSERT 引用。文件只存一份"长什么样",节省空间。

### 块属性(Attribute / ATTRIB)——提取时往往盯它

块定义里可以嵌入**可填写的文字字段**。每次 INSERT 时填具体值。

钻孔块的典型定义:
```
块定义 "钻孔":
  - 一个圆 (几何固定)
  - 固定标签 "钻号:" (文字)
  - 属性 ATTRIB "孔号"  ← 占位符,引用时填值
  - 属性 ATTRIB "孔深"  ← 占位符,引用时填值
```

→ **对自动提取**:钻孔的"位置 + 编号 + 深度"全在 INSERT 里。主要读 **位置 + 属性字典**,几乎不用看几何细节。

伪逻辑:
```
遍历所有实体
  → 筛选 "类型 == INSERT" 且 "块名 == 钻孔"
  → 读出 插入位置(x,y) + 属性字典{"孔号":"ZK-12","孔深":"350"}
  → 输出 CSV 行
```

### 煤矿图常见块(示例)

| 块名(示例) | 装什么 | 提取重点 |
|---|---|---|
| 钻孔 | 钻孔符号 | 位置 + 孔号/孔深 |
| 测点 | 测量控制点 | 位置 + 点号 |
| 工作面标记 | 工作面符号 | 位置 + 工作面编号 |
| 断层符号 | 断层标记 | 位置 + 倾角/落差 |
| 图签 | 标题栏 | 图名、比例、绘图日期等 |

## 五、坐标系与单位(坑最多的地方)

### A. 两种空间:模型空间 vs 图纸空间

| 名称 | 用途 | 比例 | 数据在哪 |
|---|---|---|---|
| **模型空间**(Model Space) | 1:1 按真实尺寸画图 | 1:1 | ⭐ 真正的几何数据 |
| **图纸空间**(Paper Space / Layout) | 排版打印,塞图框、视口 | 缩放视口 | 主要是排版元素 |

> **自动提取只看模型空间。** ezdxf 里 `doc.modelspace()` 就是它。

### B. 两套坐标系:WCS vs UCS

- **WCS(世界坐标系)**:整张图的绝对坐标系,固定不变
- **UCS(用户坐标系)**:画图人临时定义的局部坐标系,只是画图工具

DXF 文件里大部分实体的坐标已经被换算成 WCS 存进去了。

> **自动提取基本只认 WCS。** 唯一例外是 INSERT(见下条)。

### C. ⚠️ INSERT 的"局部坐标"陷阱

> **块定义里所有实体的坐标,是相对于"块自己的原点"的,不是世界坐标。**

要算出块内某个实体在图上的真实位置,必须做坐标变换:

```
真实坐标 = 块内坐标 →[缩放]→ →[旋转]→ →[平移到 INSERT 插入点]→
```

**例子**:块定义里圆心坐标 (10, 0),INSERT 插入到 (1000, 500),缩放 2,旋转 90°:
- (10, 0) 缩放 2 → (20, 0)
- (20, 0) 旋转 90° → (0, 20)
- 平移到 (1000, 500) → **真实坐标 (1000, 520)**

**好消息**:ezdxf 提供 `insert.virtual_entities()`,自动展开为已变换好的虚拟实体,不用手算。

**但必须知道这件事的存在**,否则会犯错:直接读出 INSERT 里某个圆的坐标 (0, 0),误以为这是图上位置——错。

口诀:**"块内的坐标是相对的,不是世界坐标。要靠 INSERT 的位置/旋转/缩放算出真坐标。"**

### D. 单位:DXF 坐标"没有单位"

> **DXF 文件里所有坐标都只是数字(float),不附带单位。**

DXF 头里有 `$INSUNITS` 变量声明单位,但很多图没填或填错,不能完全依赖。

**怎么判断单位?三个手段一起用:**

1. **看 $INSUNITS**(参考用)
2. **看坐标数量级**(最实用)
   - 0~10000 → 多半 mm 或 cm
   - 6 位数(如 X=3950000) → 多半是 m,可能是地方平面坐标
3. **问画图的人**(最稳)

**煤矿图常见单位:**

| 图类 | 常见单位 |
|---|---|
| 巷道平面图、地形图、采掘工程图 | **米(m)** |
| 设备零件图、机械加工图 | **毫米(mm)** |
| 地质剖面图 | 多为 米,纵向可能有夸张比例 |

### E. 煤矿图特有的"地方坐标系"问题

煤矿图常用**平面直角坐标系**(不是经纬度):

- 北京54 坐标系
- 西安80 坐标系
- CGCS2000 坐标系(国家现行标准)
- **矿区独立坐标系**(原点和北方向自定义,最坑)

→ 提取出来的 (x, y) 数字本身没意义,**必须知道它在哪个坐标系下**,才能后续在 GIS 里叠加、转经纬度、与卫星图对齐。

→ **工程级流水线里,坐标数字必须搭配"坐标系元数据"一起存。**

→ 这部分在第 13 节(pyproj 坐标转换)展开,可能派生"GIS 基础"独立方向。

## 六、自动提取的"侦察阶段"(实战补充)

> 真实工程里,没有上帝告诉你"钻孔 = INSERT,块名是'钻孔'"。每张陌生图必须自己侦察。

**完整流水线:**

```
[1] 侦察阶段(每接到一种新图都要做)
     ↓ 搞清楚:有哪些图层?有哪些实体类型?有哪些块?
[2] 制定规则
     ↓ 比如"钻孔 = 图层为'钻孔' + 类型为 INSERT + 块名为 ZK"
[3] 编写提取代码
     ↓ 按规则跑
[4] 验证 & 输出
```

第 1 步用 ezdxf 跑"摸底脚本",输出类似:

```
图层列表(共 8 个): 0, 巷道中线, 巷道边线, 断层, 钻孔, ...
实体类型统计:
  LINE: 342, LWPOLYLINE: 187, TEXT: 95, INSERT: 46, ARC: 12
INSERT 用到的块名(共 5 种):
  ZK(32 次), GZK(8 次), 测点(4 次), 图签(1 次), 指北针(1 次)
```

人对照报告 + 实际图纸,把规则总结出来,固化为代码。

→ 第 03 节(ezdxf 入门)会教如何跑出这种报告。
→ 第 09 节(自定义属性提取策略与映射规则)会教如何系统化制定规则。

## 八、关键术语速查表

| 术语 | 一句话定义 |
|---|---|
| Entity(实体) | CAD 图里一个独立的可见图形单元 |
| Layer(图层) | 实体的分类标签,可批量控制可见性/颜色 |
| Block Definition(块定义) | 可复用的图形"图章" |
| INSERT(块引用) | 在图上"盖一次章",实体类型名就叫 INSERT |
| ATTRIB(块属性) | 块定义里预设的可填写文字字段 |
| Model Space(模型空间) | 1:1 真实尺寸的画图空间,几何数据真正所在地 |
| Paper Space(图纸空间) | 排版打印用的版式空间 |
| WCS(世界坐标系) | 整张图的绝对坐标系 |
| UCS(用户坐标系) | 画图人临时定义的局部坐标系(画图工具) |
| $INSUNITS | DXF 文件头里声明单位的变量(参考用,不一定准) |
