# 02- DXF 文件结构剖析

> 本节目标:打开 DXF 文件本身看里面长什么样、按什么规则组织——GROUP CODE/值对、SECTION 框架、5 大核心段各装什么。这是后续 ezdxf 操作的"底层地基"。

## 一、DWG vs DXF

### DWG —— AutoCAD 的私有二进制格式
- **二进制**:记事本打开是乱码
- **私有**:Autodesk 私有格式,**官方规范不公开**
- **效率高**:文件小,CAD 软件读写快
- **第三方读取难度极大**:需要逆向工程,有版权风险

### DXF —— 公开的"交换格式"
- **文本格式(ASCII)**:记事本可以打开 ✅
- **公开规范**:Autodesk 出了完整文档,免费下载
- **任何编程语言都能解析**:纯文本,无门槛

### 对比表

| | DWG | DXF |
|---|---|---|
| 格式 | 二进制 | **纯文本 / ASCII** |
| 规范是否公开 | ❌ 不公开 | ✅ 完全公开 |
| 文件大小 | 小 | 大(2~5 倍) |
| 软件打开速度 | 快 | 慢 |
| 第三方读取难度 | 很难 | 很简单 |

### 自动提取的标准路线

```
拿到 .dwg 文件
   ↓ 用 ODA File Converter (免费工具) 转
.dxf 文件
   ↓ 用 ezdxf 等 Python 库读
结构化数据(几何 + 属性)
```

> **做自动提取,永远在 DXF 这一层操作**。第 10 节会专门讲 DWG ↔ DXF 转换工具链。

## 二、DXF 的基本字节单元——GROUP CODE / 值对

### 核心规则:所有内容"两行一组"

```
第 1 行: 一个整数,叫 GROUP CODE(组码)
第 2 行: 这个组码对应的值
```

### 真实片段:一条直线

```
  0
LINE              ← 组码 0:实体类型
  5
A1B               ← 组码 5:句柄(唯一 ID)
  8
巷道中线          ← 组码 8:图层
 62
1                 ← 组码 62:颜色号
 10
100.0             ← 组码 10:起点 X
 20
50.0              ← 组码 20:起点 Y
 30
0.0               ← 组码 30:起点 Z
 11
200.0             ← 组码 11:终点 X
 21
80.0              ← 组码 21:终点 Y
 31
0.0               ← 组码 31:终点 Z
```

→ 本质是"键值对"形式描述一切。

### 核心 GROUP CODE 速查

| 组码 | 含义 |
|---|---|
| **0** | 实体类型(LINE / CIRCLE / TEXT / INSERT 等) |
| **2** | 名字(图层名、块名等) |
| **5** | 句柄(实体唯一 ID,16 进制字符串) |
| **8** | 实体所在图层名 |
| **10, 20, 30** | 主要点的 X, Y, Z 坐标(起点/圆心/插入点) |
| **11, 21, 31** | 次要点的 X, Y, Z 坐标(终点) |
| **62** | 颜色号(1~255 整数) |
| **40** | 主要尺寸(半径、长度、字高等) |
| **50** | 角度(旋转角、起始角等) |
| **1** | 文本内容字符串 |

**规律**:
- `10, 11, 12, ...` 主点/次点/第三点的 X
- `20, 21, 22, ...` 对应的 Y
- `30, 31, 32, ...` 对应的 Z
- → 看到坐标族 `1x, 2x, 3x` 一组 = 一个点的 XYZ

### 读片段练习:圆

```
  0
CIRCLE
  8
钻孔
 10
500.0
 20
300.0
 30
0.0
 40
25.0
```
→ 图层"钻孔"里,圆心 (500, 300, 0),半径 25。

## 三、DXF 整体骨架——若干 SECTION 组成

### 最小 DXF 骨架

```
  0
SECTION
  2
HEADER          ← 文件头:全局设置
... 内容 ...
  0
ENDSEC

  0
SECTION
  2
TABLES          ← 各种"表":图层表、线型表等
... 内容 ...
  0
ENDSEC

  0
SECTION
  2
BLOCKS          ← ⭐ 所有"块定义"
... 内容 ...
  0
ENDSEC

  0
SECTION
  2
ENTITIES        ← ⭐⭐⭐ 模型空间所有实体(主战场)
... 内容 ...
  0
ENDSEC

  0
SECTION
  2
OBJECTS         ← 非图形对象
... 内容 ...
  0
ENDSEC

  0
EOF             ← 文件结束标志
```

### 规律
- 每个 SECTION 用 `0/SECTION` 开头,`0/ENDSEC` 结尾
- SECTION 名字在第二行(`2/HEADER`、`2/TABLES` ...)
- 整个文件最后 `0/EOF`

### 5 大核心 SECTION 一句话

| SECTION | 装什么 | 提取关心程度 |
|---|---|---|
| **HEADER** | 全局设置:DXF 版本、$INSUNITS、图纸边界 | 偶尔(看单位) |
| **TABLES** | 图层表、线型表、字体表等"目录" | 偶尔(看图层定义) |
| **BLOCKS** | ⭐ 所有块定义("图章"长什么样) | 经常(处理 INSERT 时) |
| **ENTITIES** | ⭐⭐⭐ 模型空间所有可见实体 | **几乎全部时间在这里** |
| **OBJECTS** | 字典、XRecord 等非图形对象 | 进阶才碰 |

## 四、核心 SECTION 详解

### ⭐⭐⭐ ENTITIES(实体段)—— 主战场

模型空间里所有实体:LINE、LWPOLYLINE、CIRCLE、TEXT、INSERT、DIMENSION 等。

```
  0
SECTION
  2
ENTITIES
  0
LINE          ← 第 1 个实体
  8
巷道中线
 10
100.0
...
  0
LWPOLYLINE    ← 第 2 个实体
  8
巷道边线
...
  0
INSERT        ← 第 3 个实体(块引用)
  8
钻孔
  2
ZK            ← 引用的块名
 10
500.0
 20
300.0
...
  0
ENDSEC
```

→ 提取本质 = **从 ENTITIES 段遍历实体,按规则筛**。

### ⭐ BLOCKS(块定义段)——处理 INSERT 时必看

所有块定义——也就是 INSERT 引用的那些"图章"长什么样。

```
  0
SECTION
  2
BLOCKS
  0
BLOCK
  2
ZK              ← 块名
 10
0.0
 20
0.0
  0
CIRCLE          ← 块内部实体:圆
 10
0.0
 20
0.0
 40
5.0
  0
ATTDEF          ← 块内部:属性定义(占位符)
  2
HOLE_ID
...
  0
ENDBLK
  0
BLOCK           ← 下一个块定义
...
```

**关键关联**:
```
ENTITIES 段里:
  INSERT (块名="ZK", 插入位置=(500,300))
                  │
                  │ 引用 ↓
                  ▼
BLOCKS 段里:
  BLOCK "ZK"
    → 内部 CIRCLE + ATTDEF
```

→ 处理块时需要"两段穿梭"。ezdxf 封装了跳转,但**底层是这么回事**必须知道。

### ⚠️ 重要分清:ATTDEF 在 BLOCKS,ATTRIB 在 ENTITIES

**这是 DXF 最容易卡的细节,影响代码逻辑:**

```
BLOCKS 段里:
  BLOCK "ZK"
    CIRCLE
    ATTDEF  (属性"定义",占位符 "这里以后会有'孔号'")
       ↑
       只是模板说明,没有具体值
  ENDBLK

ENTITIES 段里:
  INSERT  (块名=ZK, 插入位置=(1200,800))    ← 一个钻孔
  ATTRIB  (Tag="HOLE_ID", Value="ZK-12")    ← 这个钻孔的孔号值
  ATTRIB  (Tag="DEPTH",   Value="350")      ← 这个钻孔的孔深值

  INSERT  (块名=ZK, 插入位置=(1500,900))    ← 另一个钻孔
  ATTRIB  (Tag="HOLE_ID", Value="ZK-13")    ← 它的孔号
  ATTRIB  (Tag="DEPTH",   Value="380")      ← 它的孔深
```

总结:
- **BLOCKS 段里只有 ATTDEF(属性定义/占位符)**——像 Word 表单的"姓名:___"
- **ENTITIES 段里 INSERT 后紧跟 ATTRIB(属性值)**——填进格子的"姓名:张三"
- **每个 INSERT 各自带自己的一组 ATTRIB**——所以不同钻孔孔号可以不同

**修正后的钻孔提取思路:**
```
Step 1: 遍历 ENTITIES 段
Step 2: 筛选 (类型==INSERT) && (图层==钻孔) && (块名==ZK)
Step 3: 对每个匹配的 INSERT:
        - 读插入位置 (10, 20, 30)
        - 读它后面跟着的 ATTRIB → 取"孔号""孔深"等值
        - 这一切都在 ENTITIES 段内完成,不用去 BLOCKS!
```

**BLOCKS 段什么时候才用?** 当需要"展开块内部几何"时——比如想知道钻孔块画的那个圆的真实坐标(用 `virtual_entities()`)。

> **口诀:块的"几何模板"在 BLOCKS;块的"属性值"跟着 INSERT 一起在 ENTITIES。**

### TABLES(表段)——偶尔需要

各种"目录式"信息的定义。最重要的子表:

- **LAYER 表**:图层定义(名、颜色、线型、可见/锁定状态)
- **LTYPE 表**:线型定义(实线、虚线、点划线的图样)
- **STYLE 表**:文字样式定义
- **BLOCK_RECORD 表**:块的索引

**用场景**:侦察阶段列出所有图层清单;查图层颜色/可见性。

ezdxf 里 `doc.layers` 就对应这里。

### HEADER(头段)——看全局设置

全局变量,以 `$` 开头。最有用:

| 变量 | 含义 |
|---|---|
| `$ACADVER` | DXF 版本号(AC1024=AutoCAD 2010, AC1032=AutoCAD 2018) |
| `$INSUNITS` | 单位代码(1=英寸, 4=毫米, 6=米...) |
| `$EXTMIN` / `$EXTMAX` | 模型空间最小/最大坐标,圈出图纸范围 |

**用场景**:判断单位、判断图纸范围。

### OBJECTS(对象段)——进阶才碰

字典、扩展记录(XRecord)等"非图形对象"。一般实体提取用不到,处理复杂自定义属性时才涉及。第 8 节(XData / XRecord)展开。

## 五、提取开发者的思考路径

```
拿到一张图,想提取某类东西
       ↓
Step 1: 进 ENTITIES 段,遍历实体
       ↓
Step 2: 按 "类型 + 图层" 筛
       ↓
Step 3: 是 INSERT? → 同段读 ATTRIB 取属性值
                  → 需要展开块内几何? → 去 BLOCKS 段(或用 virtual_entities)
        不是 INSERT? → 直接读几何参数
       ↓
Step 4: 需要全局信息(单位/范围)? → 查 HEADER
        需要图层颜色/状态? → 查 TABLES.LAYER
       ↓
Step 5: 输出结构化数据
```

**时间分布大概:**
```
ENTITIES 段:  ████████████████████  80%
BLOCKS 段:    ████                  15%
HEADER 段:    █                      3%
TABLES 段:    █                      2%
OBJECTS 段:                          罕见
```

## 六、本节核心打包

| 点 | 要点 |
|---|---|
| 格式选择 | DWG 二进制+闭源,DXF 文本+开放,**提取永远走 DXF** |
| 字节单元 | 所有内容**两行一组**:GROUP CODE + 值 |
| 关键组码 | 0=类型 / 8=图层 / 2=名字 / 10,20,30=主点 / 11,21,31=次点 / 40=尺寸 / 50=角度 |
| 整体骨架 | 若干 SECTION,各以 `0/SECTION` 开头 `0/ENDSEC` 结尾,文件 `0/EOF` 结束 |
| 5 大 SECTION | HEADER / TABLES / BLOCKS / **ENTITIES**(主战场) / OBJECTS |
| 块两段穿梭 | BLOCKS=模板(ATTDEF) ;ENTITIES=实例(INSERT + ATTRIB 值) |

## 七、关键术语速查表

| 术语 | 一句话定义 |
|---|---|
| GROUP CODE(组码) | DXF 里每条信息的"键",一个整数 |
| SECTION(段) | DXF 文件的顶层组织单元,每段装一类信息 |
| HEADER | 装全局变量的段 |
| TABLES | 装图层表/线型表等"目录"的段 |
| BLOCKS | 装所有块定义("图章")的段 |
| ENTITIES | 装模型空间所有可见实体的段(提取主战场) |
| OBJECTS | 装非图形对象的段 |
| ATTDEF | 块定义里的属性占位符,在 BLOCKS 段 |
| ATTRIB | 块引用上填好的属性值,在 ENTITIES 段紧跟 INSERT |
| 句柄(Handle) | 实体的唯一 ID,16 进制字符串,组码 5 |
| EOF | DXF 文件结束标志 |
